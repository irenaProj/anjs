AngularJS Application Structure

AngularJS apps follow the MVC pattern and thus consist of 3 parts:

  • Model - application data, including support for CRUD operations on data via RESTful services.
  • View - data and logic to present the data to user.
  • Controllers - business logic - operations on model based on user interactions, including preparing data for the view.

Angular Module

Module is a container for different app parts: controllers, services, filters, etc. In scalable applications it is recommended that app will be broken into modules like this:

  • A module for each feature
  • A module for each reusable component (especially directives and filters)
  • And an application level module which depends on the above modules and contains any initialization code.
To set the scope of a module use the ng-app attribute.

Creation vs Retrieval

angular.module('myModule', [])
- will create the module myModule and overwrite any existing myModule. 

angular.module('myModule')
- will retrieve an existing myModule

Configuration and Run Blocks

                        angular.module('myModule', []).
                            config(function(injectables) { // provider-injector
                              // This is an example of config block.
                              // You can have as many of these as you want.
                              // You can only inject Providers (not instances)
                              // into config blocks.
                            }).
                            run(function(injectables) { // instance-injector
                              // This is an example of a run block.
                              // You can have as many of these as you want.
                              // You can only inject instances (not Providers)
                              // into run blocks
                            });
When bootstrapping, first Angular applies all constant definitions. Then Angular applies configuration blocks in the same order they were registered.

A run block is the code which needs to run to kickstart the application. It is executed after all of the services have been configured and the injector has been created.

Angular Scope

scope is an object that refers to the application model. It is an execution context for expressions. Scopes are arranged in hierarchical structure which mimic the DOM structure of the application. Scopes can watch expressions and propagate events.

Scopes provide context against which expressions are evaluated. For example {{username}} expression is meaningless, unless it is evaluated against a specific scope which defines the username property.

Each Angular application has exactly one root scope, but may have several child scopes.

Scopes can propagate events in similar fashion to DOM events. The event can be broadcasted (ng-click="$broadcast('MyEvent')") to the scope children or emitted (ng-click="$emit('MyEvent')") to scope parents.

Scopes and Directives

There are 2 types of directives:

  • Observing directives, such as double-curly expressions {{expression}}, register listeners using the $watch() method.
  • Listener directives, such as ng-click, register a listener with the DOM. When the DOM listener fires, the directive executes the associated expression and updates the view using the $apply() method.
Directives that create scope:

Directives, such as ng-controller and ng-repeat, create new child scopes and attach the child scope to the corresponding DOM element.

Angular Controller

In Angular, a Controller is a JavaScript constructor function that is used to augment the Angular Scope. When a Controller is attached to the DOM via the ng-controller directive, a new child scope will be available as an injectable parameter to the Controller's constructor function as $scope.

Use controllers to:

  • Set up the initial state of the $scope object
  • Add behavior to the $scope object
The most common way to keep Controllers slim is by encapsulating work that doesn't belong to controllers into services and then using these services in Controllers via dependency injection.

In the hierarchy of scopes that inherit from each other, the $scope that each Controller receives will have access to properties and methods defined by Controllers higher up the hierarchy.

Angular Service

Angular service are objects that are wired together via Dependency Injection. Services are:

  • Lazily instantiated - instantiated on use
  • Singletons - reference to the single instance generated by the service factory
Examples of built-in services: $http, $location.

Create service: 
                        angular.module('myModule', []).
                            factory('myService', function() {
                                return function() {
                                     // Perform service operation
                                };
                            });

                        // or
                        angular.module('myModule', []).config(['$provide', function($provide) {
                            $provide.factory('serviceId', function() {
                                return function() {
                                     // Perform service operation
                                };
                            });
                        }]);

Angular Filter

A filter formats the value of an expression for display to the user.

Filter with parameters: 

                        {{ expression | filter:argument1:argument2:... }}

Built-in filters: 

                        // Format the number 1234 with 2 decimal points
                        {{ 1234 | number:2 }}

Use filter in controller: 

                        angular.module('FilterModule', []).
                            controller('FilterController', ['filterFilter', function(filterFilter) {
                              this.array = [
                                {name: 'Tobias'},
                                {name: 'Jeff'},
                                {name: 'Brian'},
                                {name: 'Igor'},
                                {name: 'James'},
                                {name: 'Brad'}
                              ];
                              this.filteredArray = filterFilter(this.array, 'a');
                            }]);

Custom Filter

                    angular.module('myApp', [])
                    .filter('myFilter', function() {
                      return function(param1, param2) {
                        return out;
                      };
                    })

Custom Directives

Directives are markers on a DOM element (such as an attribute, element name, comment or CSS class) that tell AngularJS's HTML compiler ($compile) to attach a specified behavior to that DOM element or even transform the DOM element and its children.

Built-in directives examples: ngBind, ngModel ngClass.

Creating Directive

Directives are registered on modules using module.directive API. 

                        angular.module('myApp', [])
                        .controller('Controller', ['$scope', function($scope) {
                            ...
                        }])
                        .directive('useTemplate', function() {
                          return {
                            restrict: 'A',
                            templateUrl: 'template.html'
                          };
                        });
The restrict specifies:
  • 'A' - match attribute name
  • 'E' - match element name
  • 'C' - match class name
  • 'AEC' - match either attribute, element or class

To associate outer scope with directive's inner scope, create isolate scope.

Example: 
                    angular.module('docsIsolateScopeDirective', [])
                    .controller('Controller', ['$scope', function($scope) {
                      $scope.naomi = { name: 'Naomi', address: '1600 Amphitheatre' };
                      $scope.igor = { name: 'Igor', address: '123 Somewhere' };
                    }])
                    .directive('myCustomer', function() {
                      return {
                        restrict: 'E',
                        scope: {
                          customerInfo: '=info'
                        },
                        templateUrl: 'my-customer-iso.html'
                      };
                    });
Template content: 
                    Name: {{customerInfo.name}} Address: {{customerInfo.address}}

The scope option is an object that contains a property for each isolate scope binding. In this case it has just one property:

  • Its name (customerInfo) corresponds to the directive's isolate scope property customerInfo.
  • Its value (=info) tells $compile to bind to the info attribute.

Another option is to specify transclude on directive: 

                        angular.module('docsTransclusionExample', [])
                        .controller('Controller', ['$scope', function($scope) {
                          $scope.name = 'Tobias';
                        }])
                        .directive('myDialog', function() {
                          return {
                            restrict: 'E',
                            transclude: true,
                            scope: {},
                            templateUrl: 'my-dialog.html'
                            }
                          };
                        });

RESTFul Services

From Wikipedia: Representational State Transfer (REST) is an architecture style or design pattern used as a set of guidelines for creating web services which allow anything connected to a network to communicate with one another via HTTP. RESTful web services are assumed to transfer data in XML and/or JSON format. HTTP commands are GET, POST, PUT, DELETE etc.
